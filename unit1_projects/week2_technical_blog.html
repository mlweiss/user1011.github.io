CSS Positioning: Getting Your Absolutes and Relatives Straight

In this blog post I'll explain the possible positioning options available in CSS.

  1. position:static

    2. This is the default positioning for all elements in CSS. This means that the element has no specific directions for how to be positioned so it is placed as it would normally occur in the document.

    This would generally be used if it is necessary to reset a positioning that has already been set. 

    #top-box {
	position:static;
}
#bottom-box {
	position:static
}
    id = "top-border"
    id = "top-box"
    id = "bottom-box"
    id = "bottom-border"

  2. position:relative

    3. Specifying a relative position allows you to move the element relattive to where it would normally occur in the document, i.e. relative to its static position. 
    #top-box {
	position: relative;
	top: 40px;
	right: -20px
}
    id = "top-border"
    id = "top-box"
    id = "bottom-box"
    id = "bottom-border"

    Notice in the diagram how the position of top-box has changed relative to its position in the previous example. The next element did not move because its position in the flow of the document was left unchanged.

  3. position:absolute

    4. Specifying an absolute position removes the element from the document and allows it to be placed in an exact location on the page. 
    #top-box {
	position: absolute;
	top: 1210px;
	right: 50px;
}
    id = "top-border"
    id = "top-box"
    id = "bottom-box"
    id = "bottom-border"

    The remaining elements get stacked in the order that they appear in the document. Thus the bottom-box takes the position previously held by the top-box as it is not the next element in the order of the flow of the document.

  4. position:absolute + relative

    5. The power of the relative and absolute position features is clear when they are used together. By placing box-1a inside of box-1 and giving the latter box a relative position we are able to adjust the position of box-1a relative to the position of box-1. This is a bit counterintuitive as the box that remains in place is actually given the relative designation, but if you think that the absolute attribute removes the box from the document and positions it relative to the box designated relative it makes a bit more sense. 

    #box-1 {
	position: relative;
}

#box-1a {
	position: absolute;
	top: 0px;
	right: 200px;
}
    id = "top-border"
    id = "box-1"
    id = "box-1a"
    id = "bottom-border"

Conclusion

This covers some of the most useful position attributes. It would be useful to check out resorces for some of the ones not covered here, like the float and fixed positions.
